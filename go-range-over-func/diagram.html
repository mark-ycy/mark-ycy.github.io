Go Range Over Func 原理图

// Go 1.23+ · GOEXPERIMENT=rangefunc (1.22) · Iter Protocol

✦ Go 1.23 正式特性 · iter.Seq[V] / iter.Seq2[K,V]
iterator.go Iterator 函数 
// iter.Seq[V] 类型定义
type Seq[V any] func(
    yield func(V) bool,
)

// 自定义迭代器
func IntsTo(n int) iter.Seq[int] {
    return func(yield func(int) bool) {
        for i := 0; i < n; i++ {
            // 调用 yield 传递值
            if !yield(i) {
                return // break 信号
            }
        }
    }
}
main.go 调用方 
// range 遍历 iterator
for v := range IntsTo(5) {
    fmt.Println(v)
}

// 等价展开（编译器做的事）
IntsTo(5)(func(v int) bool {
    fmt.Println(v)
    return true // continue
})

// break → yield 返回 false
for v := range IntsTo(100) {
    if v > 3 { break }
    fmt.Println(v)
}

yield 机制
Iterator 通过调用 yield(v) 向 range 循环体传递值。yield 是一个普通函数，不是协程，运行在同一 goroutine。
yield(v) → bool
🛑
break / return
当循环体执行 break 或 return，编译器生成的 yield 函数返回 false，Iterator 收到信号后应停止并 return。
yield(v) == false → return
🔁
continue
循环体执行 continue 时，yield 返回 true，Iterator 继续执行，生成下一个值并再次调用 yield。
yield(v) == true → 继续
🔃
Pull vs Push 模型
Range-over-func 是 Push 模型：Iterator 主动推送值。iter.Pull() 可转换为 Pull 模型，使用 next() 按需拉取。
next, stop := iter.Pull(seq)

pull_model.go iter.Pull — Push 转 Pull 
// iter.Pull 将 Push 模式转换为 Pull 模式（内部使用 goroutine + channel 或 coro）
next, stop := iter.Pull(IntsTo(10))
defer stop()          // 必须调用！防止 Iterator goroutine 泄漏

for {
    v, ok := next()   // 主动拉取下一个值
    if !ok { break }  // 迭代结束
    fmt.Println(v)
}

// iter.Pull2 适用于 Seq2[K, V] (类似 map 的 key-value 迭代器)
